Java基础-反射

JAVA
JAVA
发布日期:   2022-03-01

Java安全的基础可以从反序列化漏洞开始说起，反序列化漏洞又可以从反射开始说起。

– Java安全漫谈（Phith0n）

Java反射（Reflection）是Java中非常重要的动态特性，Java是静态语言，而通过反射，我们可以给予java动态特性。

什么是静态语言、动态特性我们在此不提。

先列举几个反射中很重要的方法：

  • forName（可以获得这个类）
  • newInstance（可以实例化这个类）
  • getMethod（可以获得一个对象的方法）
  • invoke（可以执行一个对象的方法）

基本上，这几个方法就可以执行Java安全中绝大多数与反射相关的payload了。

有安全基础的可能就能有点感觉了，从获取类到执行方法一套攻击方式都出来了。

从一个实例来探究反射过程

//在此有个类，我们需要通过反射去调用test类的getId方法。
package com.gungnir;

public class test {
    private String name;
    private Integer id;

    public String getName() {
        return name;
    }

    public Integer getId() {
        return id;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void setId(Integer id) {
        this.id = id;
    }
}

1.获得类的实例

获得类实例上面提到了可以用forName方法。

但其实并不只有这个。

  1. 调用java.lang.Class类的静态方法forName获取某个类

    Class clazz = Class.forName(“com.gungnir.test”);

  2. 访问某个对象的getClass()方法

    Class clazz = test.getClass();

  3. 如果你已经记载了某个类，想获取它的对象，直接拿它的class属性就行，但此方法不是反射。

    Class clazz = test.class;

2.获得setId()方法

前面说了获得类方法是用getMethod

clazz.getMethod(方法名，这个方法的参数类型)

例：
Method method = clazz.getMethod("setId",Integer.class);

3.调用setId()方法

调用方法用的是invoke()

这个方法有两个参数

第一个参数是可以两种类型的

  • 如果这个方法是一个普通方法，那么第一个参数是类对象
  • 如果这个方法是一个静态方法，那么第一个参数是类

第二个参数就是你要传给方法的参数。

例：
method.invoke(test,1)

结合起来我们的流程就是

test里面是定义的类，test2里是利用反射注入了id。

可以看到我们成功注入了1，输出的就是1.

java.lang.Runtime

在反射中相关的payload中，我们经常能看到java.lang.Runtime类的身影，这是因为java.lang.Runtime类中存在exec方法，可以执行本地命令。

但其实我们不能直接这样做

Class clazz = Class.forName("java.lang.Runtime");
clazz.getMethod("exec", String.class).invoke(clazz.newInstance(),"calc.exe");

因为Runtime的构造方法是私有的。

Tips:

newInstance的作用是实例化一个类，也可以说是调用无参构造函数

在编写漏洞利用方法的时候，newInstance有时会报错，原因可能是以下两种

  • 使用的类没有无参构造函数。
  • 使用的类的构造函数是私有的。

为什么会是私有的，这就涉及到了单例模式，此类仅需要创建一个实例，将构造函数设置为私有，可以防止多次创建，那我们怎么拿到这个实例。

Runtime提供了一个静态方法来获取实例，Runtime.getRuntime()

再重新改造一下invoke后的参数就好了。

Class clazz = Class.forName("java.lang.Runtime");
clazz.getMethod("exec", String.class).invoke(clazz.getMethod("getRuntime").invoke(clazz),"calc.exe");

成功弹出计算器，执行了命令。

Java反射机制总结

Java反射机制是Java动态性中最为重要的体现，利用反射机制我们可以轻松的实现Java类的动态调用。Java的大部分框架都是采用了反射机制来实现的(如:Spring MVC、ORM框架等)，Java反射在编写漏洞利用代码、代码审计、绕过RASP方法限制等中起到了至关重要的作用。
